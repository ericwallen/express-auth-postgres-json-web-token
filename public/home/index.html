Getting Started

The sign up and log in forms above will work once you follow a few steps. Ready?

This is a shorter version documentation of the docs you can find on the github project. The more detailed version can be found on the github Read Me or the README.md in the root of this package you just spun up.


Step 1 - Install Dependencies

This is a node package with many dependencies. Assuming you have node installed run the following command whilst inside your projects root directory

npm install

Step 2 - Create a .env file and add token

This program a "environment variable". In order to set that up you would create a .env file and put a token inside. It migt look like this TOKEN_SECRET=new_hotness. You can name the value new_hotness whatever you want.

touch .env
TOKEN_SECRET=new_hotness

Step 2 - Create a Database

For this package to work out of the box you will want to name your database exampledb. If you want to change that name look in the root of the package for a file called knexfile.js. Inside that you will see the path to postgresql://localhost/exampledb. Change that to whatver your heart desires. The command you will use to create the db will look like this.

createdb exampledb

Step 3 - Migrate & Seed Database

All the migration and seed files are ready to go. All you have to do is run the commands below. Below is a ERD diagram of how the schema will be laid out unless you change it before running the commands

knex migrate:latest
knex seed:run

Log in to some of these accounts to make sure everything is working. You will see the users data from a dasboard as well as a secret specific to each one. Ater you are done doing that then you can create a new account for yourself and log in to that.
